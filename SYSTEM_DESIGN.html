01 — What is this?

Executive Overview

Freight companies process hundreds of invoices monthly. Manual auditing is slow, error-prone, and expensive. CPA AI Agent replaces that process with a fully automated AI pipeline.

What it does

Accepts PDF invoices, extracts structured data using LLMs, benchmarks freight rates against live market data, flags anomalies, and generates audit reports — all without human input.

💡

Why it exists

Invoice fraud and billing errors in freight logistics cost companies 3–5% of revenue annually. Traditional auditing tools require templates. This system works on any invoice format.

🚀

Key innovation

Combines Docling (PDF table preservation) + Groq LLM (semantic extraction) + Apify (real-time market benchmarks) in a LangGraph state machine — no brittle regex, no templates.

Why LangGraph (not simple function calls)?

LangGraph models the pipeline as a directed graph with typed state. This gives:

  • Conditional routing — non-invoices are skipped entirely, not passed through every agent
  • Shared state — AgentState dict flows between nodes, no spaghetti parameter chains
  • Visual debuggability — the graph can be rendered and inspected
  • Industry standard — used in production LLM workflows at scale

Why Docling + LLM (not regex)?

Invoice formats vary wildly between vendors. Two-step extraction wins:

  • Docling — converts PDFs to Markdown while preserving table structure (line items stay intact)
  • Groq LLM — reads semantic meaning ("this is the total") regardless of layout position
  • Caching — both Docling parse and LLM response cached by file hash, so re-runs are instant
  • Zero templates — works on any vendor's invoice without configuration
04 — Low Level Design

Component Deep Dive

Each agent is an isolated class with a single responsibility. No business logic leaks between layers.

IngestionAgent
Scans input_docs for PDFs. Sends each to Groq with a classification prompt — is this an invoice? Returns a boolean + confidence score.
ingestion.py
ExtractionAgent
Docling converts PDF → Markdown. Groq LLM extracts: vendor, invoice_number, date, currency, incoterms, line_items[], total_amount, tax, payment_terms.
extraction.py
StorageAgent
Validates the extracted InvoiceData Pydantic model. Checks for duplicate invoice numbers. Writes to SQLite via SQLAlchemy ORM. Returns StorageResult with the DB row ID.
storage.py
BenchmarkingAgent
For each line item, queries Apify for real-time freight market rates on that route. Calculates % deviation. Falls back to mock rates when USE_MOCK_APIFY=true.
benchmarking.py
AnalysisAgent
Compares benchmarked rates against BENCHMARK_THRESHOLD_PERCENT (default 15%). Classifies severity: Critical (>50%), High (>30%), Medium (>15%), Low. Generates AnomalyReport.
analysis.py
FeedbackAgent
Stores human corrections (field_name, original_value, corrected_value) in the feedback table. Tracks most common correction patterns for model improvement over time.
feedback.py

Frontend Architecture

No React. Vite bundles plain TypeScript with multi-page output. Why?

  • src/api.ts — typed API client, all 18 endpoints, single source of truth for response shapes
  • src/dashboard.ts — CSS section switching (no router), lazy-loads data per section
  • SSE Streaming — EventSource connects to /api/pipeline/stream, logs print in real-time
  • Zero framework overhead — ships 35KB of JS. React would add ~150KB for no UX gain here

Error Handling Strategy

Multi-layer resilience baked in:

  • Pydantic validation — all API inputs/outputs validated at the boundary, never trust raw data
  • Retry decorator — utils/retry.py wraps LLM calls with exponential backoff
  • Mock fallback — Apify unavailable? USE_MOCK_APIFY=true uses static rate tables, pipeline never blocks
  • Cache — Docling + LLM results cached by file hash; re-processing an invoice is instant
05 — Technology Choices

Tech Stack & Rationale

Every technology choice was deliberate. Here is what was chosen and why — not just what.

Layer Technology Category Why This, Not an Alternative
Backend FastAPI Framework Async-first, auto OpenAPI docs, Pydantic-native. Flask lacks async; Django is overkill for a pure API layer.
Agent Orchestration LangGraph AI State machine model enables conditional routing (skip non-invoices) and shared typed state. LangChain alone is linear; LangGraph handles branching.
LLM Groq + Llama 3.3-70B AI Groq's inference hardware delivers ~300 tokens/sec — 10× faster than OpenAI at <10% the cost. 70B model accuracy is sufficient for structured extraction.
PDF Parsing Docling (IBM) Processing Preserves table structure as Markdown. PyPDF2/pdfminer flatten tables into garbled text, destroying line-item data. Docling is the only OSS tool that solves this reliably.
Market Data Apify Scraping Managed scraper infrastructure for freight rate websites. Eliminates maintaining custom scrapers that break when sites change their HTML.
Database SQLite + SQLAlchemy Storage Zero-configuration, file-based, sufficient for thousands of invoices. PostgreSQL would add operational complexity with no benefit at this scale. SQLAlchemy ORM enables a future DB swap with one config change.
Validation Pydantic v2 Types Runtime type enforcement on all LLM outputs and API boundaries. LLMs hallucinate fields — Pydantic catches bad responses before they reach the database.
Frontend Build Vite + TypeScript Frontend Sub-second HMR, multi-page static output, Rollup bundling. No React needed — the dashboard is a data-display app, not an interactive SPA.
Styling Tailwind CSS (CDN) Frontend Utility-first, zero CSS file management. CDN mode avoids a build step for styling; custom design tokens via config for the "Sovereign Auditor" dark theme.
Real-time SSE (Server-Sent Events) Protocol Pipeline logs stream to the dashboard in real-time. WebSockets are bidirectional (overkill for one-way log streaming); SSE is simpler, HTTP-native, and auto-reconnects.
Package Manager uv (Astral) Tooling 10–100× faster than pip. Rust-based resolver, lock-file based reproducibility. Docling's heavy dependencies (torch, CUDA) install in seconds, not minutes.
Containerization Docker (multi-stage) DevOps Builder stage installs deps (8GB+ with torch/CUDA); runtime stage copies only the .venv and app code. Reproducible deploys regardless of host OS.
Config Pydantic Settings Config Reads from .env + environment variables with type validation at startup. App fails fast with a clear error if GROQ_API_KEY is missing, not silently at first LLM call.

CI/CD Flow

git push → GitHub → Render/Railway detects push → rebuilds Docker image → deploys → health check on /api/health → live.

🛡️

Security

All secrets in env vars. CORS configured. File upload validates MIME type + extension. No secrets ever committed to git (.env in .gitignore).

📈

Scalability Path

SQLite → PostgreSQL: one DATABASE_URL change. File storage → S3: one adapter change. LangGraph nodes → async tasks: swap StateGraph for distributed queue.
